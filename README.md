# Поздравления

Проект MVP сервиса групповых открыток и капсулы поздравлений.

## Документы проекта

- `docs/WORKING_AGREEMENT.md` — рабочие правила проекта и обязательные инженерные требования.
- `docs/MVP_PRODUCT_BRIEF.md` — зафиксированная версия продуктового фокуса MVP 1.0.
- `docs/TEMPLATE_SYSTEM.md` — система 4 базовых шаблонов для первой версии.
- `docs/UI_BLOCK_ARCHITECTURE.md` — архитектура универсальных визуальных блоков и шаблонов.
- `docs/GIT_REMOTE_SETUP.md` — что нужно для подключения GitHub и будущих push.
- `docs/DECISIONS_LOG.md` — журнал продуктовых и технических решений.
- `docs/DELIVERY_LOG.md` — журнал того, что сделано, что отложено и какие новые идеи появились.

## Базовый принцип

Сначала держим фокус на MVP 1.0:

- сценарии `учитель`, `воспитатель`, `коллега`;
- один тариф;
- сильный финальный опыт получения открытки;
- AI-помощник для написания поздравления;
- обязательные тесты и логирование.

## Текущий стек

- `Next.js`
- `TypeScript`
- `Vitest`
- `ESLint`
- встроенное структурированное логирование на уровне приложения

## Текущее состояние

- `Блок 1` завершен: каркас, тесты, логирование, стартовые страницы.
- `Блок 2` доведен до рабочего состояния: есть форма создания открытки, серверная валидация, выбор шаблона и локальное сохранение черновика.
- `Блок 3` доведен до рабочего состояния: есть публичная ссылка участника, форма поздравления и локальное сохранение поздравлений.
- `Блок 4` доведен до рабочего состояния: есть AI-помощник для черновиков поздравлений, лимиты генераций и логирование AI-сценария.
- `Блок 5` доведен до рабочего состояния: есть секретная ссылка организатора, управление поздравлениями и базовый предпросмотр.
- Зафиксирована block-based архитектура финальной открытки: шаблоны теперь понимаются как универсальные визуальные форматы, а не как жесткие сценарные макеты.
