## Админка

Доступна по адресу `/admin`. Перед использованием нужно задать учётные данные администратора.

1. Сгенерировать хеш пароля:
   ```bash
   node scripts/generate-admin-password.mjs
   ```
2. Добавить значения в `.env.local`:
   ```bash
   ADMIN_EMAIL=admin@example.com
   ADMIN_PASSWORD_HASH=<hash из скрипта>
   ADMIN_PASSWORD_SALT=<salt из скрипта>
   ADMIN_SESSION_SECRET=<secret из скрипта>
   ```
3. Применить миграции (если используется PostgreSQL):
   ```bash
   set -a && source .env.local && set +a && npm run db:migrate
   ```
4. Перейти на `/admin/login` и войти.

Возможности Phase 1:
- Dashboard со сводкой по открыткам, поздравлениям и фото.
- Список открыток с фильтрами и сменой статуса.
- Детальная страница открытки.
- Модерация поздравлений.

Возможности Phase 2:
- Раздел «Заказы и оплаты» для ручного управления статусами заказов.
- Раздел «Шаблоны» — редактирование названий, описаний, акцента и включение/выключение шаблонов.
- Раздел «Администраторы» — создание пользователей с ролями admin/moderator/support.
- Ролевая модель: support видит Dashboard и поздравления, moderator — ещё и открытки, admin — полный доступ.
- При логине сначала проверяется `admin_users`, затем fallback на env-переменные.

## Базовый принцип

Сначала держим фокус на MVP 1.0:

- сценарии `учитель`, `воспитатель`, `коллега`;
- один тариф;
- сильный финальный опыт получения открытки;
- AI-помощник для написания поздравления;
- обязательные тесты и логирование.

## Текущий стек
