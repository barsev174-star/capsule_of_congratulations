# Анимация открытия подарочной открытки

Дата актуализации: 2026-06-29.

## Статус

Вступительная анимация для `/gift/[finalSlug]` реализована и локально проверена. API, модель данных и содержимое финальной открытки не менялись.

## Сценарий

1. Получатель видит закрытый конверт, имя и кнопку `Открыть открытку`.
2. После нажатия печать исчезает, клапан открывается вверх и остаётся позади открытки.
3. Сложенная открытка становится видна внутри конверта: нижняя часть скрыта передним карманом.
4. Открытка поднимается перед конвертом.
5. Открытка раскрывается снизу вверх и приближается к экрану.
6. На обеих внутренних половинах показывается реальный выбранный шаблон.
7. Анимационная карточка плавно сменяется полноценной страницей открытки.

## Детали поведения

- В закрытом конверте вложенная открытка полностью скрыта.
- Оборот оформлен как фирменная карточка `Дари слова` с эмблемой и лёгким растительным декором.
- Анимация запоминается в `sessionStorage` и не повторяется при обычном обновлении в той же сессии.
- После просмотра доступна кнопка повторного открытия.
- Кнопка `Пропустить` сразу показывает финальную открытку.
- При `prefers-reduced-motion: reduce` используется сокращённый переход.
- Неопубликованная открытка показывает спокойный экран ожидания.
- Параметр `?forceIntro=1` работает только в development и предназначен для визуальной проверки.

## Проверено

- Desktop viewport: закрытый конверт, пауза после клапана, подъём, раскрытие и переход.
- Mobile viewport: посадка конверта и открытки не выходит за экран.
- Настоящий шаблон полностью помещается по ширине; его шапка не обрезается.
- Ошибок браузера нет.
- `npm run build` проходит.
