# Минимальная аналитика и мониторинг ошибок

Дата: 2026-07-06.

## События пользовательского пути

- `funnel.card_creation_started` — пользователь начал создание открытки;
- `funnel.card_created` — черновик создан;
- `funnel.participant_link_copied` — скопирована ссылка участника;
- `funnel.participant_form_opened` — участник открыл форму;
- `funnel.contribution_submitted` — поздравление сохранено;
- `funnel.card_published` — открытка опубликована;
- `funnel.gift_opened` — получатель открыл подарок.

События записываются как JSON в стандартный вывод приложения и дополнительно сохраняются для административной сводки. В production используется PostgreSQL (`telemetry_events`, миграция `0010_telemetry_events.sql`), локально без `DATABASE_URL` — `data/telemetry-events.json`. Локальный файл не входит в Git.

Идентификатор `cardId` позволяет собрать воронку одной открытки без хранения имени, email, текста поздравления или секретных ссылок. Хранилище ограничено 10 000 последними событиями в локальном JSON; административная выборка ограничена выбранным периодом и 10 000 строками.

## Критические ошибки

События `critical.database`, `critical.publication`, `critical.media`, `critical.email`, `critical.ai` содержат безопасный контекст и уникальный `errorId`. Этот же код показывается пользователю там, где это помогает поддержке найти сбой.

Глобальные ошибки браузера отправляются как `client.unhandled_error`. Endpoint `/api/telemetry` принимает только известные события и поля `cardId`, `source`, `route`, `component`.

Если запись телеметрии недоступна, пользовательское действие не блокируется: основной сценарий продолжается, а сбой сохранения телеметрии остаётся в структурированном логе.

## Экран администратора

Защищённая страница `/admin/analytics` доступна только роли `admin` и показывает:

1. количество событий, уникальных открыток и критических ошибок;
2. воронку пользовательского пути и конверсию между соседними шагами;
3. периоды 7 и 30 дней;
4. последние критические ошибки с `errorId` и `cardId`.

Данные начинают накапливаться после подключения этого слоя; исторические Docker-логи в таблицу автоматически не импортируются.

## Просмотр на VPS

Примеры поиска в логах контейнера:

```sh
docker logs capsule-web --since 24h 2>&1 | grep '"event":"funnel.'
docker logs capsule-web --since 24h 2>&1 | grep '"event":"critical.'
docker logs capsule-web --since 24h 2>&1 | grep '<errorId>'
```

Тексты поздравлений, имена, email, токены, одноразовые ссылки и сырые ответы AI в телеметрию не записываются. Оплата этим слоем не затрагивается.
