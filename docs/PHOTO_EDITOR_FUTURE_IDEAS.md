# Фото открытки: будущие улучшения

Этот список не входит в текущую мобильную доработку. Сейчас цель — сделать добавление фото и подписи удобными на узком экране, без масштабной переработки хранения и модели данных.

## Надёжность серверной части

- Сделать сервер единственным источником истины для выбора блока: выбирать `GREETINGS`, затем первый доступный блок по стабильному `sortOrder`.
- Учитывать при выборе блока его видимость, права редактирования, лимит блока, общий лимит открытки и состояние шаблона.
- Ввести стабильные доменные коды ошибок для лимитов, недоступности блока, конфликтов загрузки и файлового хранилища.
- Проверять лимиты в транзакционном участке, чтобы параллельные загрузки не превышали их.
- Перевести файл в постоянное состояние только после создания записи в БД; временные файлы очищать при ошибке или через существующий TTL-механизм.

## Подписи

- Считать длину по grapheme clusters через общий helper (`Intl.Segmenter`), а не через `string.length`.
- Согласовать один и тот же лимит и нормализацию подписи на клиенте и сервере.
- В зависимости от блока показывать фактический допустимый лимит без автоматического обрезания текста.

## Управление фото

- Ввести порядок фотографий внутри каждого блока с кнопками «Назад» и «Вперёд» для мобильных устройств.
- При переносе фото в другой блок добавлять его в конец нового списка и безопасно пересчитывать порядок старого.
- Добавить доступное управление меню и порядком: корректный фокус, `aria-controls`, `aria-expanded` и сообщения для скринридера.
- Рассмотреть замену файла в уже добавленной карточке только вместе с отдельной поддержкой на backend.

## Проверки

- Добавить интеграционные тесты конкурентной загрузки, rollback временного файла и очистки незафиксированных объектов.
- Добавить визуальные проверки для ширин 360, 375, 390 и 430 px, а также для desktop 1024–1600 px.
