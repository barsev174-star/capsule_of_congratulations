# MVP-flow после первичного дизайна

Дата: 2026-06-22

## Продуктовый фокус

Первичный визуальный дизайн scrapbook-открытки зафиксирован. Следующий этап — не дальнейшая полировка, а рабочий жизненный цикл одной открытки:

1. организатор создает открытку;
2. получает ссылку управления;
3. отправляет участникам ссылку сбора поздравлений;
4. участники добавляют поздравления;
5. организатор модерирует, выбирает главное поздравление и проверяет финальную версию;
6. получатель открывает финальную ссылку.

## Что уже есть

1. Стартовый экран и создание открытки.
2. Экран управления и модерации.
3. Предпросмотр.
4. Финальная открытка.
5. Страница участника.
6. Scrapbook-шаблон с настроенным визуальным конфигом.

## Что добавлено в MVP-flow

1. Статусы открытки:
   - `draft` — черновик;
   - `collecting` — сбор поздравлений;
   - `ready` — готова к отправке;
   - `closed` — сбор закрыт.
2. Панель ссылок на экране организатора:
   - участникам;
   - организатору;
   - получателю.
3. Копирование ссылок из панели организатора.
4. Закрытие сбора поздравлений через статус `closed`.
5. Базовые действия в финале:
   - реакция “Спасибо”;
   - сохранение через печать/системное сохранение страницы;
   - переход к созданию новой открытки.
6. Ограничение медиа: до 6 фото на одну открытку.

## Что сознательно отложено

1. Регистрация и личные кабинеты.
2. Полноценная админка сервиса.
3. Оплата.
4. AI как обязательная часть сценария.
5. Экспорт PNG/PDF.
6. Отправка реальной реакции организатору.
7. Постоянная база данных и облачное хранилище файлов.

## Решение по фото

На ближайший MVP в одной открытке может быть максимум 6 фото:

1. до 3 фото рядом с блоком поздравлений;
2. до 3 фото в блоке “Моменты”.

Вертикальный вариант блока поздравлений использует один слот вместо горизонтальной группы, но общий лимит библиотеки всё равно остается 6 фото. Это упрощает хранение, ускоряет загрузку открытки и делает объём данных на VPS предсказуемым.

## Следующие разумные шаги

1. Перенести хранение из JSON в устойчивую БД.
2. Подключить файловое хранилище для пользовательских изображений.
3. Подготовить деплой.
4. Добавить базовые error/empty states для устаревших и закрытых ссылок.
5. После проверки ручного flow подключать AI точечно:
   - улучшить поздравление;
   - сократить поздравление;
   - выделить качества;
   - выбрать лучшие фразы.
