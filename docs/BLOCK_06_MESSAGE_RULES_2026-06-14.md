# Block 6: жесткие правила блока поздравлений

Актуальный статус проекта после первичного дизайна и MVP-flow см. в `docs/PROJECT_HANDOFF_2026-06-23.md`. Этот файл остается исторической заметкой по правилам поздравлений.

Дата: `2026-06-14`

## Что сделали

1. Вынесли правила раскладки поздравлений в отдельный слой.
2. Для каждого режима задали собственные ограничения:
   - сколько карточек помещается в экран
   - сколько символов безопасно помещается в одну карточку
   - каким шагом должен происходить переход
3. Подключили эти лимиты к валидации:
   - при добавлении поздравления участником
   - при редактировании поздравления организатором
4. Перевели горизонтальные режимы на перелистывание без “обрезанных” карточек.
5. Для `Сетки 2 на 2` сделали переход по одному ряду.
6. Для `Двух рядов` сделали переход по одной колонке.
7. Для `1 колонка + фото` закрепили медиаблок справа, а поздравления перевели на вертикальную прокрутку слева.
8. На странице участника добавили живой счетчик символов и сохранили текст в форме при ошибке.
9. AI-помощник теперь заранее знает лимит карточки и старается выдавать варианты, которые помещаются в выбранный формат.

## Почему это важно

1. Финальная открытка больше не должна показывать половину следующего поздравления.
2. У каждого режима теперь понятная и предсказуемая логика чтения.
3. Участник сразу видит, сколько текста еще помещается.
4. AI перестает провоцировать ошибку “сначала сгенерировал, потом не дал сохранить”.

## Что еще требует живого теста

1. Насколько комфортен реальный сценарий чтения в `Два ряда` на длинных списках.
2. Достаточно ли лимита `220` символов для режима `1 колонка + фото`.
3. Когда именно расширенную сетку `2x3` стоит включать автоматически, если вокруг мало других блоков.
4. Нужен ли отдельный “суперблок” для длинных поздравлений, вместо дальнейшего увеличения обычных карточек.
