Нужно доработать landing page сервиса «Дари слова».

Важно:
карточки контентных блоков не делать в виде цельных изображений.
Их нужно реализовать нативно в интерфейсе: HTML/CSS/React components.

### Не генерировать как картинки:
- блок «Для каких случаев»
- блок «Что будет внутри открытки»
- блок «Как это работает»
- блок преимуществ
- FAQ-карточки

### Генерируемыми / декоративными ассетами остаются только:
- hero illustration
- декоративные элементы CTA
- небольшие фоновые / paper / envelope / floral accents
- при необходимости preview-иллюстрации шаблонов

---

## 1. Блок «Для каких случаев»

Сделать отдельную секцию:
Заголовок: «Для каких случаев»
Подзаголовок: «Подходит почти для любого повода, когда хочется сказать тёплые слова вместе.»

Сетка:
- desktop: 6 карточек в ряд
- tablet: 3 в ряд
- mobile: 2 в ряд или 1 в ряд, в зависимости от ширины

Карточки:
- фон: бело-кремовый / paper-light
- border: тонкий тёплый
- radius: 20–24px
- padding: 20–24px
- лёгкая тень
- hover: subtle lift + чуть более заметная тень
- внутри:
  - иконка сверху
  - title ниже
  - без лишнего текста или максимум одна короткая строка

Сценарии:
1. День рождения коллеги
2. Учителю или воспитателю
3. От всей команды
4. Для друга или подруги
5. Юбилей
6. Увольнение / переход / благодарность

Иконки:
- использовать аккуратные svg-иконки в едином стиле
- тёплые оттенки, не синие
- soft orange / terracotta / peach / muted pink / warm gold
- без кислотных цветов

---

## 2. Блок «Что будет внутри открытки»

Сделать отдельную секцию:
Заголовок: «Что будет внутри открытки»
Подзаголовок: «Всё, что делает подарок живым и личным.»

Карточки должны быть кодом, не изображением.

Сетка:
- desktop: желательно в одну строку, 7 карточек
- если не помещается аккуратно — сделать:
  - либо 7 компактных карточек в одну строку,
  - либо CSS grid с auto-fit,
  - но визуально стремиться к one-row layout на широком desktop
- tablet/mobile — перенос на 2–3 ряда

Элементы:
1. Обложка с именем
2. Поздравления от участников
3. Фото и моменты
4. Главное поздравление
5. Лучшие фразы
6. За что тебя ценят
7. Анимация открытия

Каждая карточка:
- маленькая иконка сверху
- title
- короткое описание в 1–2 строки
- мягкий paper card style
- одинаковая высота
- аккуратная внутренняя типографика

Важно:
заголовки и иконки не должны уходить в коричневый.
Основной accent — оранжевый / тёплый terracotta.
Коричневый допустим только как основной текстовый цвет, но не как акцент интерфейса.

---

## 3. Иконки для карточек

Иконки для блоков не генерировать отдельными растровыми ассетами.
Нужно использовать:
- либо lucide/react-icons/heroicons,
- либо собственные svg-компоненты,
- но привести их к единому стилю.

Стиль иконок:
- лёгкие
- rounded
- без тяжёлых чёрных контуров
- в тёплой палитре
- можно поместить в мягкую подложку-кружок / squircle

Размер:
- 20–24px сама иконка
- контейнер 40–48px

---

## 4. Блок шаблонов

Карточки шаблонов тоже должны быть вёрсткой, а не одной большой картинкой.
Если реальных ассетов шаблонов пока нет:
- сделать временные preview placeholders,
- но структура карточки должна быть готова под замену ассета позже.

Карточка шаблона:
- preview area сверху
- title
- short description
- мягкий hover
- состояние selected / recommended при необходимости

Подготовить компонент так, чтобы позже можно было просто подставить:
- image
- title
- description
- accent color

---

## 5. Подготовка к будущим ассетам

Нужно сразу заложить структуру, чтобы позже можно было легко заменить заглушки на реальные ассеты.

Для этого:
- у hero, CTA corners и template previews сделать props / data fields под image src
- если ассета нет — показывать placeholder
- если ассет появится позже — просто подставлять изображение без переписывания layout

---

## 6. Визуальные требования

Стиль всей страницы:
- warm handmade greeting card aesthetic
- cream paper background
- serif headings
- orange / terracotta accent
- мягкие карточки
- много воздуха
- не должно быть ощущения SaaS dashboard

Важно:
- не использовать коричневый как accent color для иконок, счётчиков и интерактивных элементов
- accent = orange / terracotta
- текст = тёмно-коричневый
- secondary text = muted warm brown

## 7. Что нужно на выходе

Нужно:
1. Доработать секцию «Для каких случаев» как набор UI-карточек
2. Доработать секцию «Что будет внутри открытки» как набор UI-карточек
3. Привести иконки в единый тёплый стиль
4. Подготовить структуру под будущие ассеты
5. Не ломать текущую вёрстку и адаптивность

Если где-то сейчас используются растровые временные карточки, заменить их на нормальные React/UI-компоненты.
