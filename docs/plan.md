Нужно доработать нижнюю часть страницы `/join`: блок “Так будет выглядеть открытка” и блок “Есть повод скоро?”.

Цель: после добавления поздравления участник должен понять ценность сервиса и захотеть либо посмотреть пример открытки, либо оставить дату будущего повода. Блоки должны выглядеть тёпло, просто и не как техническая форма.

Важно:

* не менять основную форму поздравления;
* не менять AI-блок;
* не менять блок “Уже добавили”, кроме отступов при необходимости;
* не менять бизнес-логику отправки поздравления;
* работать только с нижними CTA-блоками на `/join`;
* сохранить стиль `/join`: cream/peach, orange primary, rounded cards, scrapbook/paper feel;
* build должен пройти.

---

## 1. Блок “Так будет выглядеть открытка”

### 1.1 Переименовать блок

Сейчас текст:

```text
Так будет выглядеть ваша открытка
```

Заменить на:

```text
Так получатель увидит общий подарок
```

Почему: участник не владелец открытки, он добавляет поздравление в общий подарок.

### 1.2 Label

Текущий label:

```text
ПОДАРОК ЦЕЛИКОМ
```

Оставить или заменить на более мягкий:

```text
ГОТОВАЯ ОТКРЫТКА
```

Предпочтительно:

```text
ГОТОВАЯ ОТКРЫТКА
```

### 1.3 Описание

Заменить описание на:

```text
Ваше поздравление станет частью красивой страницы с тёплыми словами, фото и поздравлениями от всех.
```

Текст должен быть короче и эмоциональнее. Не использовать длинное “после публикации получатель увидит...”.

### 1.4 Кнопка

Кнопка:

```text
Посмотреть пример открытки →
```

Требования:

* secondary/outline style;
* orange text/border;
* не слишком мелкая;
* на mobile full width.

### 1.5 Визуал справа

Сейчас визуал хороший по идее, но немного перегружен и зажат.

Нужно:

* сделать визуал крупнее на 10–15%;
* композицию упростить;
* главная карточка открытки должна быть читаемой;
* фото должны быть второстепенными;
* не делать слишком много мелких элементов;
* визуал должен выглядеть как preview готовой открытки, а не как рекламный баннер.

Рекомендуемый состав:

* основная мини-открытка;
* 1–2 фото справа/сзади;
* мягкая тень;
* лёгкий наклон;
* без перегруза.

### 1.6 Layout

Desktop:

```text
left: текст + CTA
right: preview открытки
```

Mobile:

```text
preview
текст
CTA
```

На mobile визуал не должен занимать слишком много высоты.

## 2. Блок “Есть повод скоро?”

### 2.1 Убрать label “СЕКРЕТНЫЙ ПОВОД”

Сейчас label:

```text
СЕКРЕТНЫЙ ПОВОД
```

Заменить на:

```text
НАПОМНИМ ЗАРАНЕЕ
```

Он понятнее и лучше объясняет функцию.

### 2.2 Заголовок

Оставить:

```text
Есть повод скоро?
```

### 2.3 Описание

Заменить текст на:

```text
Оставьте дату — мы напомним, когда будет время собрать такую же открытку.
```

Смысл: без регистрации, без тяжёлого обещания, просто напоминание.

---

## 3. Упростить форму напоминания

Сейчас форма выглядит слишком плотной и немного технической.

### 3.1 Поля

Оставить только:

```text
Кого поздравить?
Повод
Дата
Email
```

Если сейчас есть select “Как напомнить?” и доступен только Email — убрать это поле или скрыть до появления других каналов.

Не нужно показывать выбор канала, если выбор фактически один.

### 3.2 Placeholder

```text
Кого поздравить? — Например, маму
Повод — День рождения
Дата — ДД.ММ.ГГГГ
Email — name@example.com
```

### 3.3 Кнопка

```text
Напомнить мне
```

Требования:

* orange primary;
* шире и заметнее;
* на desktop в одну линию с email/date, если помещается;
* на mobile full width.

### 3.4 Helper text

Добавить короткий helper под формой:

```text
Пришлём письмо заранее, чтобы вы успели собрать открытку от всех.
```

Если есть согласие на данные, оставить чекбокс, но сделать текст тише и компактнее.

---

## 4. Визуально сделать блок теплее

Сейчас блок “Есть повод скоро?” выглядит как форма. Нужно добавить ощущение подарка.

Слева можно оставить маленький подарок/конверт, но сделать его:

* чуть крупнее;
* мягче;
* не как иконку, а как маленький декоративный объект.

Карточка:

* background warm white/cream;
* мягкая border;
* лёгкая тень;
* меньше плотности;
* больше воздуха вокруг заголовка.

---

## 5. Порядок блоков

Порядок оставить:

```text
1. Уже добавили
2. Так получатель увидит общий подарок
3. Есть повод скоро?
4. Нижняя плашка про то, что слова попадут в открытку
```

Это правильный порядок: сначала человек видит участие других, потом результат, потом предложение вернуться.

## 7. Mobile

Проверить mobile:

* блок example не должен быть слишком высоким;
* preview не должен обрезаться;
* кнопка full width;
* reminder form в одну колонку;
* поля не должны быть слишком мелкими;
* нет горизонтального скролла.
