# Доработки `/manage` перед подготовкой оплаты

Дата актуализации: 2026-07-06.

Статус: выполнено локально. Production deploy в рамках этой задачи не выполнялся.

## Выполнено

1. Кнопка `Сохранить изменения` в блоке основы открытки закреплена справа на desktop независимо от наличия текста статуса.
2. Пробелы по краям полей больше не создают ложное состояние `Есть несохранённые изменения`; после успешного сохранения форма синхронизируется с нормализованными сервером значениями.
3. На mobile скрыта иконка drag-and-drop для блоков открытки, а кнопки перемещения вверх/вниз расположены вертикально.
4. `Бумажный классический` установлен шаблоном по умолчанию для новых открыток.
5. Исправлена AI-генерация в ручном добавлении поздравления: кнопка `Получить 3 варианта` больше не отправляет внешнюю форму, не перезагружает страницу и не возвращает пользователя на вкладку `Оформление`.
6. Блок приглашения участников ранее переработан: полный URL скрыт, основное действие копирует join-ссылку, форма участника открывается отдельно.
7. Блок доступа организатора переименован в `Ваш доступ к открытке`, добавлены понятное описание и подсказка о 15 минутах и отсутствии рассылок.
8. Обычное сохранение не отправляет access email, если адрес не менялся.
9. При добавлении или изменении email отправляется одно письмо и показывается соответствующий результат.
10. Добавлена вторичная кнопка `Отправить ссылку ещё раз` с состояниями успеха, лимита и ошибки.

## Проверки

- `npm test`: 260 тестов прошли, 3 пропущены.
- Targeted ESLint для изменённых TypeScript-файлов: проходит.
- `npm run build`: production-сборка и TypeScript проходят.
- Общий ESLint по проекту по-прежнему содержит ранее существовавшие ошибки `react-hooks/set-state-in-effect` в других manager-компонентах; это отдельный технический долг.

## Что проверить при production deploy

1. Вход в `/admin` с production-учётными данными: ранее сообщалось об ошибке пароля.
2. Доставку access email после изменения адреса и по кнопке повторной отправки.
3. Полный сценарий `/manage` на desktop и реальном телефоне.
4. AI-генерацию из формы ручного поздравления с production-провайдером.
