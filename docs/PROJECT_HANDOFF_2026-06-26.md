## Текущая локальная разработка

Локально приложение использует PostgreSQL из `.env.local`:

```text
DATABASE_URL=postgres://capsule:capsule_pass@127.0.0.1:5433/capsule
```

Если появляется ошибка `ECONNREFUSED 127.0.0.1:5433`, почти всегда остановлен Docker-контейнер локальной базы.

Сценарий запуска:

```powershell
cd "C:\Project\Поздравления2.0"
docker start capsule-postgres-local
docker ps --filter "name=capsule-postgres-local"
npm.cmd run dev
```

В `docker ps` должно быть:

```text
0.0.0.0:5433->5432/tcp
```
