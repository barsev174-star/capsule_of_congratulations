## Главные проблемы на текущем скриншоте

### 1. Открытка всё ещё слишком узкая

Сейчас финальная открытка выглядит как узкая бумажная лента внутри телефона. Из-за этого:

* текст всё ещё мелковат;
* карточки выглядят сжатыми;
* фото не дают достаточного эмоционального эффекта;
* блоки кажутся длинной полосой, а не мобильной открыткой.

Нужно сделать открытку шире.

Ориентир:

```css id="81lgcb"
.mobileGiftPage {
  width: 100%;
  overflow-x: hidden;
}

.mobileGiftInner,
.giftMobileInner,
.giftPaperMobile {
  width: min(100%, 430px);
  max-width: 430px;
  margin: 0 auto;
  padding-left: 8px;
  padding-right: 8px;
  box-sizing: border-box;
}

.giftSection,
.giftPaperSection {
  width: 100%;
  box-sizing: border-box;
}
```

Важно проверить, нет ли внутри мобильной открытки фиксированной ширины вроде:

```css id="chmjgt"
width: 280px;
width: 300px;
width: 320px;
max-width: 320px;
```

Если есть — заменить на адаптивную ширину.

На телефоне открытка должна занимать примерно:

```text id="fji651"
92–96% ширины экрана
```

без горизонтального скролла.

## 2. Блок “Моменты” сломан

На скриншоте заголовок блока “Моменты” обрезан слева: видно почти только “менты”. Это критический баг.

Нужно:

* убрать отрицательные margin/translate у заголовка;
* проверить родительский контейнер;
* проверить `overflow`, `position:absolute`, `left`, `transform`;
* заголовок должен начинаться внутри контента и не уходить за левую границу.

Правильно:

```text id="k5svx4"
Моменты
```

не должно быть:

```text id="20qsj5"
...менты
```

Для заголовков всех секций на mobile:

```css id="vmdfov"
.mobileSectionTitle {
  width: 100%;
  margin-left: 0;
  padding-left: 0;
  transform: none;
  position: relative;
  left: auto;
  box-sizing: border-box;
}
```

## 3. Блок “Моменты”: фото должны быть аккуратной сеткой

Сейчас фото в “Моментах” выглядят тесно и немного хаотично.

Для 3 фото всегда показывать:

```text id="spqwny"
[одно большое фото]

[фото 2] [фото 3]
```

Требования:

* первое фото на всю ширину;
* второе и третье ниже в две колонки;
* подписи читаемые;
* фото не должны обрезаться странно;
* ничего не должно уходить за край контейнера.

Пример CSS-логики:

```css id="tckimp"
.mobileMomentsGrid {
  display: grid;
  grid-template-columns: repeat(2, minmax(0, 1fr));
  gap: 10px;
  width: 100%;
}

.mobileMomentsGrid .momentCard:first-child {
  grid-column: 1 / -1;
}

.momentCard {
  width: 100%;
  min-width: 0;
  overflow: hidden;
  box-sizing: border-box;
  border-radius: 16px;
}

.momentCard img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

Высоты:

```css id="lxf6k7"
.momentCard:first-child img {
  min-height: 190px;
}

.momentCard:not(:first-child) img {
  min-height: 125px;
}
```

## 4. Поздравления: сейчас показывается слишком много до кнопки

На скриншоте до кнопки видно больше 4 поздравлений. Нужно зафиксировать правило:

```text id="1kjs0j"
На mobile сначала показываем 4 поздравления.
После клика “Показать ещё” добавляем по 5 сообщений.
```

То есть на первой загрузке должно быть:

```text id="q7oox7"
Алексей
Марина
Игорь
Ольга

[Показать ещё поздравления]
```

Если всего поздравлений 6, кнопка должна быть:

```text id="xsdzmw"
Показать ещё 2 поздравления
```

После клика показываются оставшиеся.

Требования:

* initial visible count: 4;
* load more step: 5;
* все одобренные поздравления доступны;
* на mobile поздравления всегда идут одной колонкой;
* не показывать сразу 6, 8, 10 или 100 поздравлений без раскрытия.

## 7. Качества

Сейчас блок “Качества” близок к правильному, но нужно проверить стабильность.

У нас 5 качеств. На mobile должно быть:

```text id="26pm2p"
доброта        чувство юмора
поддержка      внимание
              лёгкость
```

Требования:

* 2 колонки;
* первые 4 качества — сетка 2×2;
* 5-е качество — по центру на отдельной строке;
* карточки не обрезаются справа;
* длинный текст переносится внутри карточки;
* сетка не должна выходить за контейнер.

```css id="jxkz7o"
.mobileQualitiesGrid {
```

## 10. Проверить обрезания и горизонтальный overflow

На текущем скриншоте видно, что некоторые элементы могут уходить за левую или правую границу. Особенно это заметно в “Моментах”.

Проверить весь mobile gift на:

```css id="wztq7o"
width: 100vw внутри контейнера с padding;
negative margin;
left/right absolute positioning;
transform translateX;
fixed width больше родителя;
overflow visible у горизонтальных элементов;
carousel styles, которые не нужны на mobile.
```

Добавить базовую защиту:

```css id="6d0l6k"
* {
  box-sizing: border-box;
}

.mobileGiftPage {
  overflow-x: hidden;
}

.mobileGiftPage img,
.mobileGiftPage button,
.mobileGiftPage section,
.mobileGiftPage article {
  max-width: 100%;
}
```

## Acceptance criteria

После правки:

* открытка на mobile выглядит шире, не как узкая бумажная полоска;
* нет горизонтального скролла;
* ничего не обрезается слева или справа;
* заголовок “Моменты” отображается полностью;
* блок “Моменты” отображает 3 фото как 1 большое + 2 ниже;
* блок с 3 фото в поздравлениях отображает 1 большое + 2 ниже;
* блок с 2 фото отображается столбиком;
* на первой загрузке показывается ровно 4 поздравления;
* кнопка “Показать ещё” добавляет по 5 поздравлений;
* все одобренные поздравления доступны;
* качества из 5 элементов отображаются 2×2 + 5-й по центру;
* текст читается без приближения;
* desktop не ломается;
* build проходит.
