# ТЗ для Kimi: оживление демонстрационной страницы Slovesto

## 1. Цель

Добавить на демонстрационную страницу плавные scroll-анимации и микроинтеракции, сохранив:

* текущую структуру;
* тексты;
* размеры блоков;
* светлую бумажную стилистику;
* производительность;
* удобство на мобильных устройствах.

Страница должна ощущаться современной и технологичной, но не превращаться в яркий SaaS-лендинг.

Основной принцип:

> Пользователь прокручивает не статичное полотно, а последовательно раскрывающуюся историю открытки.

---

# 2. Что не менять

Не менять без отдельного согласования:

* порядок смысловых секций;
* текущие CTA;
* выбор двух шаблонов;
* логику выбора шаблона;
* запуск анимации конверта;
* ссылки на демонстрационные открытки;
* общую палитру;
* типографику;
* footer;
* мобильную структуру.

Не добавлять:

* горизонтальную прокрутку всей страницы;
* тяжёлые 3D-сцены;
* резкие вращения;
* полноэкранные видео;
* анимации, задерживающие доступ к кнопкам;
* постоянное движение всех декоративных элементов.

---

# 3. Технический подход

Предпочтительно использовать:

```text
IntersectionObserver
CSS transitions
CSS transform
CSS opacity
```

Если в проекте уже используется Framer Motion или Motion One, разрешено использовать существующую библиотеку.

Не подключать новую тяжёлую библиотеку только ради простых reveal-анимаций.

Создать универсальный компонент или hook:

```ts
useScrollReveal()
```

или:

```tsx
<ScrollReveal
  variant="fade-up"
  delay={120}
  once
>
  ...
</ScrollReveal>
```

Поддерживаемые варианты:

```ts
type RevealVariant =
  | "fade-up"
  | "fade-down"
  | "slide-left"
  | "slide-right"
  | "scale-in"
  | "stagger";
```

---

# 4. Общие параметры анимаций

Использовать единые токены:

```css
:root {
  --motion-fast: 240ms;
  --motion-normal: 480ms;
  --motion-slow: 720ms;

  --motion-ease-out:
    cubic-bezier(0.22, 1, 0.36, 1);

  --motion-ease-soft:
    cubic-bezier(0.25, 0.8, 0.25, 1);
}
```

Начальное смещение:

```text
fade-up:     translateY(28px)
fade-down:   translateY(-18px)
slide-left:  translateX(-42px)
slide-right: translateX(42px)
scale-in:    scale(0.96)
```

Конечное состояние:

```text
opacity: 1
transform: none
```

Не использовать смещения больше 60 px.

---

# 5. Правила запуска

Анимация секции запускается, когда в viewport попало примерно 18–25% элемента.

Рекомендуемые параметры:

```ts
threshold: 0.18
rootMargin: "0px 0px -8% 0px"
```

Анимации проигрываются один раз:

```ts
once: true
```

При прокрутке вверх элементы не должны снова исчезать и появляться.

CTA и формы должны оставаться кликабельными даже до завершения декоративной анимации.

---

# 6. Первый экран

## 6.1. Header

При первом открытии:

* логотип плавно появляется сверху;
* служебная плашка справа появляется с небольшой задержкой.

Параметры:

```text
логотип:
fade-down
duration 420ms
delay 0ms

плашка:
fade-down
duration 420ms
delay 100ms
```

Не скрывать header при прокрутке.

## 6.2. Hero-текст

Последовательность:

1. eyebrow;
2. главный заголовок;
3. описание;
4. кнопки;
5. информационные chips.

Использовать `fade-up` с каскадной задержкой:

```text
eyebrow:     0ms
заголовок:  80ms
описание:  160ms
кнопки:    240ms
chips:     320ms
```

Для chips добавить stagger:

```text
60ms между элементами
```

## 6.3. Hero-коллаж

Коллаж справа:

* появляется через `slide-right + fade`;
* продолжительность 700–800 ms;
* задержка около 180 ms.

После появления допустимо очень слабое плавающее движение:

```css
transform: translateY(-4px);
```

цикл 6–8 секунд.

Отключать это движение при:

```css
prefers-reduced-motion: reduce
```

Не двигать весь коллаж вслед за курсором.

---

# 7. Секция «Выберите пример открытки»

## 7.1. Контейнер секции

Белая карточка секции появляется через:

```text
fade-up
duration 560ms
```

Номер шага появляется немного раньше заголовка:

```text
номер: scale-in
заголовок: fade-up
описание: fade-up
```

## 7.2. Карточки шаблонов

Первая карточка:

```text
slide-left + fade
```

Вторая карточка:

```text
slide-right + fade
```

Запускать почти одновременно, но с задержкой 100–140 ms между ними.

На мобильном обе карточки должны использовать `fade-up`, без бокового движения за пределы экрана.

## 7.3. Hover

При наведении на карточку:

```css
transform: translateY(-4px);
box-shadow: слегка усилить;
```

Продолжительность:

```text
220–260ms
```

Превью внутри можно слегка увеличить:

```css
transform: scale(1.015);
```

Не увеличивать более чем до `1.025`.

## 7.4. Выбор шаблона

После выбора:

* border плавно меняет цвет;
* появляется badge `Выбрано`;
* badge использует `scale-in`;
* предыдущая карточка спокойно возвращается в обычное состояние.

Не трясти и не подпрыгивать выбранную карточку.

---

# 8. Секция «Выберите анимацию открытия»

## 8.1. Появление секции

Вся карточка:

```text
fade-up
```

Левая часть с конвертом:

```text
slide-left
```

Правая схема:

```text
slide-right
```

На мобильном — последовательный `fade-up`.

## 8.2. Мини-демонстрация конверта

При попадании секции в viewport один раз выполнить короткую декоративную анимацию:

1. закрытый конверт появляется;
2. клапан слегка приподнимается;
3. карточка выходит вверх примерно на 8–12 px;
4. возвращается в исходное превью-состояние.

Общая длительность:

```text
900–1200ms
```

Это только декоративный preview. Полная анимация по-прежнему запускается кнопкой:

```text
Запустить анимацию
```

## 8.3. Стрелка между состояниями

Стрелка в схеме справа может:

* появляться через fade;
* или плавно «дорисовываться» слева направо.

Не использовать бесконечное мигание.

---

# 9. Секция «Посмотрите открытку глазами получателя»

Это главный смысловой блок страницы, поэтому его анимация должна отличаться от предыдущих.

## 9.1. Контейнер

Секция появляется через мягкий `fade-up`.

## 9.2. Коллаж слева

Коллаж:

```text
slide-left + scale-in
duration 700ms
```

После появления отдельные бумажные элементы могут раскрыться каскадом:

```text
50–80ms между слоями
```

Не анимировать каждую мелкую фотографию отдельно, если это создаёт хаос.

## 9.3. Текст справа

Последовательность:

1. заголовок;
2. описание;
3. CTA;
4. список преимуществ.

Использовать `slide-right + fade`.

Пункты списка появляются каскадом:

```text
70ms между строками
```

Галочка каждого пункта:

```text
scale-in
```

## 9.4. Лёгкий parallax

Разрешён слабый parallax только для двух крупных декоративных слоёв коллажа:

```text
передний слой:  4–6 px
задний слой:    2–3 px
```

Не применять parallax к тексту и CTA.

На мобильном parallax отключить.

---

# 10. Нижний CTA

Блок:

```text
Хотите собрать такую же открытку?
```

появляется через:

```text
fade-up + scale-in
```

Значения:

```text
scale from 0.975 to 1
duration 520ms
```

Иконка слева появляется первой, затем текст и кнопка.

Кнопка может получить один короткий highlight после появления:

```text
очень слабое свечение или подъём на 2px
```

Только один раз. Не делать постоянную пульсацию.

---

# 11. Разнообразие прокрутки

Для визуального ритма использовать следующую последовательность:

```text
Hero:
каскад + появление справа

Секция 1:
карточки навстречу друг другу с двух сторон

Секция 2:
двухколоночное появление + короткая анимация конверта

Секция 3:
коллаж слева, текст справа, stagger списка

Финальный CTA:
scale-in
```

Не применять один и тот же `fade-up` ко всем элементам страницы.

При этом не использовать больше двух разных эффектов внутри одной секции.

---

# 12. Дополнительный scroll-progress

На desktop можно добавить компактный индикатор прохождения трёх шагов.

Вариант:

```text
● Выбор шаблона
○ Анимация
○ Результат
```

Расположение:

* фиксированно сбоку внутри основной ширины страницы;
* либо как тонкая вертикальная линия слева от блоков.

При переходе к следующему разделу активный шаг меняется.

На экранах меньше `1100px` индикатор скрывать.

Это дополнительная задача P2. Реализовывать только после основных reveal-анимаций.

---

# 13. Мобильная версия

На мобильном:

* не использовать большие горизонтальные смещения;
* заменить `slide-left/right` на `fade-up`;
* отключить parallax;
* уменьшить stagger до 40–60 ms;
* не задерживать появление CTA;
* не создавать layout shift;
* избегать анимации высоты больших секций.

Максимальная продолжительность мобильной reveal-анимации:

```text
450ms
```

---

# 14. Reduced motion

Обязательно поддержать:

```css
@media (prefers-reduced-motion: reduce)
```

В этом режиме:

* все элементы сразу видимы;
* нет parallax;
* нет плавающих декоративных объектов;
* нет автоматического открытия конверта;
* hover-переходы сокращены;
* функциональные состояния продолжают работать.

Пример:

```css
@media (prefers-reduced-motion: reduce) {
  [data-reveal] {
    opacity: 1 !important;
    transform: none !important;
    transition: none !important;
  }
}
```

---

# 15. Производительность

Анимировать только:

```text
opacity
transform
```

Избегать анимации:

```text
width
height
top
left
box-shadow с большим blur в каждом кадре
filter blur
```

Допускается статическое изменение box-shadow на hover.

Не создавать отдельный scroll listener для каждого блока.

Использовать один общий `IntersectionObserver`.

После раскрытия элемента можно прекратить его наблюдение.

Не допускать CLS при загрузке изображений:

* задавать `width` и `height`;
* использовать `aspect-ratio`;
* резервировать место под изображения.

---

# 16. Связь с главной страницей

Не переносить весь интерактив демонстрационной страницы на главную.

На главной выполнить только следующие изменения.

## 16.1. Hero

Кнопка:

```text
Посмотреть примеры
```

ведёт на демонстрационную страницу.

При необходимости переименовать:

```text
Посмотреть, как это выглядит
```

## 16.2. Блок шаблонов

Для каждой карточки шаблона добавить действие:

```text
Открыть интерактивный пример
```

При нажатии открывать демонстрационную страницу с заранее выбранным шаблоном:

```text
/example?template=paper
/example?template=route
```

Если query-параметры не используются, допускается состояние через hash:

```text
/example#paper
/example#route
```

Не создавать дублирующие demo-страницы без необходимости.

## 16.3. Лёгкая демонстрация на главной

Допускается оставить на главной только один небольшой интерактив:

* короткое открытие конверта;
* переключение двух шаблонных превью;
* CTA перехода к полной демонстрации.

Не встраивать всю страницу демонстрации внутрь главной.

---

# 17. Аналитика

Добавить события:

```text
demo_page_view
demo_template_selected
demo_animation_started
demo_card_opened
demo_create_clicked
demo_scroll_step_viewed
```

Для `demo_scroll_step_viewed` передавать:

```ts
{
  step: "template" | "animation" | "recipient_view"
}
```

Событие каждого шага отправлять один раз за просмотр страницы.

---

# 18. Acceptance criteria

Работа считается завершённой, если:

* hero появляется последовательным каскадом;
* коллаж hero появляется справа;
* две карточки шаблонов появляются с разных сторон;
* блок конверта имеет короткую preview-анимацию;
* коллаж результата появляется слева, текст — справа;
* списки раскрываются через stagger;
* финальный CTA получает отдельную scale-анимацию;
* элементы не проигрывают reveal повторно при прокрутке вверх;
* мобильная версия не имеет горизонтального overflow;
* поддерживается `prefers-reduced-motion`;
* анимируются преимущественно `opacity` и `transform`;
* не ухудшаются Core Web Vitals;
* функциональные кнопки доступны без ожидания анимации;
* главная страница ведёт к полной демонстрации, но не дублирует её;
* карточки шаблонов на главной могут открывать демо с нужным выбранным шаблоном.
