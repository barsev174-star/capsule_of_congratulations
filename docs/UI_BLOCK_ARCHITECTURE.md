# Архитектура визуальных блоков

Актуальный handoff проекта см. в `docs/PROJECT_HANDOFF_2026-06-23.md`. Этот файл описывает архитектурный слой финальной открытки.

## Главная идея

Шаблон открытки — это не один длинный экран, нарисованный вручную под конкретный повод.

Шаблон — это:

1. визуальный стиль;
2. набор допустимых блоков;
3. порядок этих блоков;
4. правила, какие блоки обязательны, а какие зависят от контента.

## Почему это лучше

Такой подход дает:

1. меньше дублирования;
2. универсальные шаблоны вместо узких “для профессии”;
3. более гибкие демо-примеры;
4. возможность усиливать продукт за счет новых блоков, а не только новых расцветок.

## Базовые блоки MVP

### 1. `hero`

Главная обложка:

1. имя получателя;
2. повод;
3. от кого открытка;
4. фото или декоративная обложка;
5. кнопка открытия поздравлений;
6. число участников.

### 2. `summary`

Короткий вводный блок:

1. “Анна глазами команды”;
2. общий абзац;
3. позже сюда может переехать AI-summary.

### 3. `qualities`

Блок “какая ты / какой ты для нас”:

1. чипсы с качествами;
2. акцент на ощущение человека глазами группы.

### 4. `messages`

Основной список поздравлений:

1. карточки участников;
2. разный визуальный ритм;
3. компактный или расширенный вариант.

### 5. `memories`

Блок воспоминаний и медиа:

1. фото;
2. подписи;
3. композиция-полароиды или более чистая галерея.

### 6. `quotes`

Блок “лучшие фразы”:

1. короткие цитаты;
2. выделение сильных формулировок;
3. визуальная пауза между большими секциями.

### 7. `closing`

Финальный эмоциональный аккорд:

1. короткое итоговое обращение;
2. CTA;
3. сохранить / поделиться / создать такую же.

## Правила обязательности блоков

Минимальный финальный экран должен иметь:

1. `hero`
2. `messages`
3. `closing`

Условные блоки:

1. `summary` — если есть хороший вводный текст
2. `qualities` — если есть собранные качества
3. `memories` — если есть фото или медиа
4. `quotes` — если есть достаточно сильные короткие фразы

## 4 универсальных визуальных формата

### 1. `warm-classic`

Логика:

1. упор на теплый тон;
2. hero + summary + qualities + messages + closing;
3. memories и quotes — по ситуации.

### 2. `team-modern`

Логика:

1. более собранный и структурный вид;
2. hero + summary + qualities + messages + quotes + closing;
3. memories — если реально есть хороший материал.

### 3. `bright-celebration`

Логика:

1. сильный вау-экран;
2. hero + messages + quotes + closing;
3. summary — короче и динамичнее.

### 4. `gentle-personal`

Логика:

1. больше воздуха и эмоции;
2. hero + summary + memories + messages + closing;
3. qualities и quotes — как усилители, а не обязательная база.

## Следствие для разработки

Дальше мы проектируем не “еще один шаблон”, а:

1. новый блок;
2. новую комбинацию блоков;
3. новый визуальный стиль поверх уже существующих блоков.
