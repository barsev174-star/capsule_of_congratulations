# Главная страница: актуальная локальная реализация

Дата: 16 июля 2026 года.

## Назначение

Главная страница Slovesto объясняет последовательность: создать открытку, открыть сбор поздравлений, при необходимости оплатить финальную подготовку и отдельно передать открытку получателю.

## Актуальные правила продукта в текстах

- Участники добавляют только поздравления.
- Фотографии добавляет организатор.
- Создание, оформление и сбор поздравлений доступны бесплатно.
- Оплата нужна для финальной подготовки и передачи открытки получателю.
- Оплата не закрывает сбор и не передаёт открытку автоматически.
- В интерфейсе не используется термин «публикация открытки».

## AI-демонстрация

На маркетинговой странице AI не вызывается и не имитируется кнопкой. Сразу показаны исходные мысли участника и три варианта результата: «Аккуратно», «Теплее» и «Живее».

## Motion

- Reveal секций и карточек работает через `IntersectionObserver` и запускается один раз.
- Header компактнее после прокрутки.
- У hero есть очень лёгкий parallax на `requestAnimationFrame`.
- Timeline прорисовывает пунктирную линию при появлении.
- При `prefers-reduced-motion: reduce` контент не скрывается, а движения отключаются.

## Связь с демонстрационной страницей (обновлено 20 июля)

- Вторичная кнопка hero — «Посмотреть, как это выглядит» — ведёт на `/example` (ранее якорь `#templates`).
- Карточки активных шаблонов (`paper-birthday`, `route-adventure`) содержат действие «Открыть интерактивный пример» → `/example?template=<id>` с предвыбранным шаблоном.
- Scroll-анимации самой демонстрационной страницы описаны в `docs/EXAMPLE_PAGE_MOTION_2026-07-20.md`.

## Проверка

`npm run lint` и `npm run build` проходят локально.

Встроенный браузер Codex в этой сессии не подключился: в установленном пакете отсутствует модуль клиента. Поэтому перед выкладкой требуется отдельная ручная визуальная проверка локальной страницы в разрешениях 1440 px, 1024 px и 390 px.

## Обновление 21 июля: доработка главной страницы

Два раунда правок по ТЗ, работа велась в существующих компонентах без новых маршрутов и feature flags.

### Шапка и мобильное меню

- Шапка закреплена (`fixed`), высота в токене `--home-header-height`; страница получила компенсирующий `padding-top`, секции — `scroll-margin-top`, поэтому появление/скрытие шапки не двигает контент.
- Скрытие при прокрутке вниз через `transform` с порогами 14/10 px, без скачков.
- Мобильное меню на `button` с `aria-expanded`, Escape, focus trap и scroll lock с компенсацией ширины scrollbar; бургер показывается до 1100 px (ранее в диапазоне 768–1100 px навигации не было).
- FAQ по умолчанию закрыт.

### Блок сравнения «Не поток сообщений, а настоящий подарок»

- Слева — слегка хаотичная лента сообщений с настоящим фото-превью; справа — миниатюра готовой открытки с коллажем. Одно фото и фразы повторяются в обеих частях.
- Локальные демонстрационные фото: `public/assets/landing/demo-chat-photo.jpg`, `demo-collage-1.jpg`, `demo-collage-2.jpg` (сгенерированы, ~35 КБ).
- Циклическое «оживление» чата (8 с, только `transform`/`opacity`) запускается только в viewport через `src/app/_home/chat-liveliness.tsx`; при reduced-motion блок статичен.

### ИИ-демонстрация

- Черновик — разговорная заметка на рабочем листе в линейку; результат — чистая тёплая карточка без видимости textarea.
- Три варианта («Аккуратно» 214, «Теплее» 209, «Живее» 203 символа) — все ≤ 240 символов, показывается один вариант за раз, высота карточки стабильна.
- Карандаш и стрелка объединены в одну композицию с разовой анимацией; активная вкладка читается (фон, терракотовый текст, weight 600, нижняя линия); правая кнопка «Выбрать вариант ♡».

### Поводы, шаблоны, оплата, CTA, футер

- Шесть предметных мини-иллюстраций поводов вместо универсальных line-icons (торт, книга с цветком, люди, полароид, шары, конверт с сердцем).
- Шаблоны: стрелка ссылки 3 px, `focus-visible`, превью-ассеты шаблонов не тронуты.
- Оплата: «399 ₽» с `white-space: nowrap`, мягче тени, единый размер галочек; структура блока и формулировка цены не изменены.
- Финальный CTA: декор уменьшен и приглушён на мобильном, чтобы не перекрывать текст.
- Футер: «Поддержка» — раскрывающаяся группа, email закреплён снаружи; плавное раскрытие через `grid-template-rows`.

### Блок «Как это работает»

- Карточки компактнее (min-height 132 px), круги связаны с карточками соединительным штрихом, нумерация усилена (28 px, 0.8 rem, тонкая рамка).
- Предметные иконки шагов (открытка с карандашом, ссылка с участниками, конверт с открыткой).
- Линия проходит точно через центры кругов на desktop и mobile; последовательная анимация «линия → номер и иконка → карточка» на CSS animations с `backwards`; после завершения постоянного движения нет.

### Исправления и проверки

- Исправлен горизонтальный скролл от входных `translateX`-анимаций (`overflow-x: clip` на секциях случаев, оплаты и ИИ).
- Исправлено исчезновение карточек «Как это работает» после анимации (возврат к `opacity: 0` из `[data-motion-ready]`).
- Регрессионная проверка `scripts/check-home-adaptive.mjs` (playwright-core + системный Edge): горизонтальный скролл на 360/390/430/768/1024/1440 px и видимость контента при `prefers-reduced-motion`. Прогон 21 июля: 0 px overflow на всех ширинах.
- `npm run lint` и `npm run build` чистые; ошибки `tsc --noEmit` встречаются только в существующих `*.test.ts(x)` (нет типов vitest в tsconfig) и к правкам не относятся.
